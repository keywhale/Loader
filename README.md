# How to Use the Loader Library

The Loader library provides a thread-safe, async-aware system for loading, accessing, saving, and deleting keyed values in a Bukkit plugin. It manages the full lifecycle of a value (loading from storage, granting access to callers, saving back to storage, deleting from storage) with correct handling of concurrent requests.

---

## Core Concepts

### Value Lifecycle

Each keyed value moves through states internally:

- **Unknown** — not in memory, no tracker
- **Loading** — being fetched from storage asynchronously
- **Active** — in memory, one or more accessors are using it
- **Unloading** — being saved back to storage asynchronously
- **Deleting** — being deleted from storage asynchronously

Requests that arrive while a value is unloading or deleting are queued and handled once the operation completes. You do not need to manage this yourself.

### Threading Model

The Loader uses the Bukkit scheduler internally:
- **Part 1 of Access** (e.g. database query) runs **async**
- **Part 2 of Access** (e.g. constructing the in-memory object) runs **sync**
- **Part 1 of Save** (e.g. serialization) runs **sync**
- **Part 2 of Save** (e.g. writing to database) runs **async**
- **Delete** runs **async**

You must respect this in your operation implementations.

## Step 1: Implement the Operation Interfaces

### `AccessOperation<ID, VAL>`

Responsible for loading a value from storage. Created fresh per access request.

```java
public class MyAccessOperation implements AccessOperation<UUID, MyData> {

    private final UUID requestedId; // the ID passed to Loader.access() — may be null if you support creation
    private UUID resolvedId;
    private MyData data;

    public MyAccessOperation(UUID requestedId) {
        this.requestedId = requestedId;
    }

    @Override
    public boolean runPart1() throws Part1AccessOperationException {
        // Runs ASYNC. Query your database here.
        // Return true if the value was found (or created), false if not found.
        MyRow row = database.query(requestedId);
        if (row == null) return false;
        this.resolvedId = row.getId();
        this.rawData = row; // store for part 2
        return true;
    }

    @Override
    public void runPart2() throws Part2AccessOperationException {
        // Runs SYNC. Build your in-memory object here.
        this.data = new MyData(this.rawData);
    }

    @Override
    public UUID id() {
        return this.resolvedId;
    }

    @Override
    public MyData value() {
        return this.data;
    }
}
```

### `SaveOperation`

Responsible for saving a value back to storage when all accessors are done. Created fresh per save.

```java
public class MySaveOperation implements SaveOperation {

    private final UUID id;
    private final MyData data;
    private byte[] serialized;

    public MySaveOperation(UUID id, MyData data) {
        this.id = id;
        this.data = data;
    }

    @Override
    public void runPart1() throws Part1SaveOperationException {
        // Runs SYNC. Serialize / prepare data here.
        this.serialized = data.serialize();
    }

    @Override
    public void runPart2() throws Part2SaveOperationException {
        // Runs ASYNC. Write to database here.
        database.save(this.id, this.serialized);
    }
}
```

> If Part 1 throws, Part 2 will be skipped automatically.

### `DeleteOperation`

Responsible for deleting a value from storage.

```java
public class MyDeleteOperation implements DeleteOperation {

    private final UUID id;

    public MyDeleteOperation(UUID id) {
        this.id = id;
    }

    @Override
    public void run() throws DeleteOperationException {
        // Runs ASYNC.
        database.delete(this.id);
    }
}
```

## Step 2: Extend `Loader`

Subclass `Loader<ID, VAL>` and implement the three factory methods. These methods must not call any methods on the Loader itself.

```java
public class MyLoader extends Loader<UUID, MyData> {

    private final MyDatabase database;

    public MyLoader(JavaPlugin plugin, MyDatabase database) {
        super(plugin);
        this.database = database;
    }

    @Override
    protected AccessOperation<UUID, MyData> opAccess(UUID identifier) {
        return new MyAccessOperation(identifier);
    }

    @Override
    protected SaveOperation opSave(UUID identifier, MyData value) {
        return new MySaveOperation(identifier, value);
    }

    @Override
    protected DeleteOperation opDelete(UUID identifier) {
        return new MyDeleteOperation(identifier);
    }
}
```

## Step 3: Use `Accessor` to Access Values

An `Accessor` is passed to `Loader.access()` and receives an `Access` object when the value is ready. It holds on to the access until it calls `access.done()`, at which point the Loader knows it is finished and may begin saving.

### `Access<ID, VAL>`

| Method | Description |
|---|---|
| `value()` | Returns the loaded value |
| `id()` | Returns the resolved ID |
| `done()` | Releases the access; Loader may save when all accessors are done |

### Simple one-shot accessor (no cancel needed)

Use `Accessor.of(Consumer)` when you just want to read or modify the value and immediately release it:

```java
loader.access(playerId, Accessor.of(access -> {
    MyData data = access.value();
    data.setCoins(data.getCoins() + 100);
    access.done(); // release immediately
}));
```

### Long-lived accessor (with cancel support)

Use `Accessor.of(Function)` when the accessor needs to hold the value for a period of time and return a cancel callback. The returned `Runnable` is called if the value is deleted or the loader shuts down while the accessor is still active.

```java
loader.access(playerId, Accessor.of(access -> {
    MyData data = access.value();
    // e.g. attach data to a player session
    PlayerSession session = new PlayerSession(data);
    activeSessions.put(playerId, session);

    // Return a cancel callback
    return () -> {
        // Called if the value is deleted or loader shuts down before done() is called
        activeSessions.remove(playerId);
        // Note: do NOT call access.done() here; the Loader handles cleanup
    };
}));

// Later, when the player quits:
PlayerSession session = activeSessions.remove(playerId);
session.save();
session.getAccess().done(); // tell the Loader you're finished
```

### Implement `Accessor` directly

For full control:

```java
public class PlayerSessionAccessor implements Accessor<UUID, MyData> {

    private Access<UUID, MyData> access;

    @Override
    public void init(Access<UUID, MyData> access) {
        this.access = access;
        // set up whatever you need
    }

    @Override
    public void cancel() {
        // called if the value is deleted or loader shuts down
        // clean up your state here
    }

    public void release() {
        this.access.done();
    }
}
```

## Step 4: Loader Public API

### `access(ID, Accessor)`

Load a value and grant access to the accessor. If the value is not found, the accessor is never initialized.

```java
loader.access(playerId, myAccessor);
```

### `access(ID, Accessor, onNotFound, onAccessException, onSaveException)`

Full overload with callbacks:

- `onNotFound` — `Runnable` called (on main thread) if the value does not exist
- `onAccessException` — `AccessOperationExceptionHandler` called if loading fails
- `onSaveException` — `SaveOperationExceptionHandler` called if saving fails

```java
loader.access(
    playerId,
    myAccessor,
    () -> player.sendMessage("Data not found."),
    exc -> plugin.getLogger().severe("Load failed"),
    exc -> plugin.getLogger().severe("Save failed")
);
```

If any exception handler is `null`, the Loader logs the error to the plugin logger by default.

### `delete(ID)`

Delete a value from storage. If the value is currently active, all accessors are cancelled first.

```java
loader.delete(playerId);
```

### `delete(ID, DeleteOperationExceptionHandler)`

Same as above with a custom error handler.

### `shutdown()`

Initiates a graceful shutdown. Saves all currently active values and rejects new `access()` calls (throwing `Loader.ShuttingDownException`). The save operations are scheduled and run asynchronously — the method returns immediately.

```java
// In onDisable():
loader.shutdown();
```

### `shutdown(SaveOperationExceptionHandler)`

Same with a custom save error handler.

### `shutdownNow()`

Calls `shutdown()` then `expedite()`. Blocks until all pending operations complete. Use this in `onDisable()` when you need everything flushed before the plugin unloads.

```java
// In onDisable():
loader.shutdownNow();
```

### `expedite()`

Runs all pending scheduled runnables synchronously on the current thread. Useful for testing or when you need to force all in-flight operations to complete immediately.

## Exception Handlers

All exception handlers have a `handleDefault(Exception)` method that is called for all exception types by default. Override the specific methods to handle particular exception types differently.

| Interface | Methods |
|---|---|
| `AccessOperationExceptionHandler` | `handlePart1(Part1AccessOperationException)`, `handlePart1(RuntimeException)`, `handlePart2(...)`, `handlePart2(RuntimeException)` |
| `SaveOperationExceptionHandler` | `handlePart1(Part1SaveOperationException)`, `handlePart1(RuntimeException)`, `handlePart2(...)`, `handlePart2(RuntimeException)` |
| `DeleteOperationExceptionHandler` | `handle(DeleteOperationException)`, `handle(RuntimeException)` |

All methods have default implementations that delegate to `handleDefault`, so you only need to override what you care about.

## Minimal Example

```java
public class MyPlugin extends JavaPlugin {

    private MyLoader loader;

    @Override
    public void onEnable() {
        this.loader = new MyLoader(this, new MyDatabase());
    }

    @Override
    public void onDisable() {
        this.loader.shutdownNow();
    }

    public void onPlayerJoin(Player player) {
        this.loader.access(
            player.getUniqueId(),
            Accessor.of(access -> {
                player.sendMessage("Welcome back, your coins: " + access.value().getCoins());
                // Hold access until the player leaves
            }),
            () -> player.sendMessage("No data found, creating fresh profile."),
            null,
            null
        );
    }

    public void onPlayerQuit(Player player) {
        // retrieve your accessor and call access.done()
    }
}
```
